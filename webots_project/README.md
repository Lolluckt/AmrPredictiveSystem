# Webots — цех з AMR (цифровий двійник)

Частина дипломного проєкту **Інформаційна система збору телеметрії та
предиктивного обслуговування**. Сцена реалізує цех 40 × 24 м із 9 зонами,
трьома автономними роботами AMR та повною симуляцією режимів деградації.

## Запуск

1. Встановити Webots R2025a (https://cyberbotics.com).
2. `File → Open World` → `worlds/factory_floor.wbt`.
3. Натиснути ▶. Роботи починають патрулювати цех одразу після warmup.

### Публікація у MQTT (для інтеграції з backend)

```bash
pip install paho-mqtt
# Увімкнення публікації:
set AMR_MQTT=1            # Windows
export AMR_MQTT=1         # Linux/Mac
# За замовчуванням broker = localhost:1883
```

## Зони цеху

| Зона | Межі (м)       | Колір |
|------|----------------|-------|
| Склад сировини           | 0..10 × 0..9     | синій         |
| Коридор (головний, NS)   | 10..13 × 0..24   | сірий (3 м)   |
| Ділянка збирання A       | 13..23 × 0..9    | зелений       |
| Коридор (поперечний, EW) | 0..40 × 9..11    | сірий (2 м)   |
| Ділянка збирання B       | 13..23 × 11..24  | зелений       |
| Зона зварювання          | 23..32 × 0..9    | бурштиновий   |
| Зона пакування           | 23..32 × 11..24  | рожевий       |
| Зона зарядки             | 0..10 × 11..15   | блакитний     |
| Контроль якості          | 0..10 × 15..24   | фіолетовий    |
| Склад готової продукції  | 32..40 × 0..24   | індиго        |

**Ключова зміна порівняно з попередньою версією** — коридори тепер 2–3 м
завширшки, обладнання прибране до стін, всі waypoints розташовані у коридорах.
Роботи більше не застрягають у кутах стелажів чи біля зарядних станцій.

## Роботи

| ID     | Колір     | Маршрут                                                         |
|--------|-----------|-----------------------------------------------------------------|
| AMR-01 | синій     | Склад → Ділянка збирання A → Ділянка збирання B                 |
| AMR-02 | помаранчевий | Збирання A → Зварювання → Пакування → Склад готової           |
| AMR-03 | зелений   | QC → Склад готової → Зарядка (замкнутий цикл з автозарядкою)    |

Кожен робот має: Ø 0.30 м, 2 двигуни + енкодери, GPS, компас, IMU,
акселерометр, гіроскоп, **360° LiDAR (32 промені)**, 8-секторний
сонарний кільцевий датчик, статус-LED.

## Ключові покращення руху

* **Планувальник шляхів** — хендкрафтовий граф вузлів у коридорах +
  BFS-пошук. Жоден маршрут не проходить крізь робочі зони.
* **Каскадне керування** — P-регулятор курсу → трансформація у швидкості
  колес, із гальмуванням за відстанню до цільової точки.
* **Злиття даних LiDAR + сонарів** — 3-рівневе уникнення перешкод
  (slow → swerve → stop-and-turn).
* **Відновлення після застрягання** — детекція `<8 см за 2.5 с` → реверс
  з коригуванням курсу; після 4 спроб waypoint пропускається (self-heal).

## Імітовані режими відмов (для Predictive Maintenance)

| Ключ                 | Ефект                                                             |
|----------------------|-------------------------------------------------------------------|
| `bearing_right`      | Зростання вібрації правого колеса + падіння ККД                   |
| `thermal_left`       | Прискорене нагрівання лівого двигуна (× 2.2)                      |
| `battery_fade`       | Прискорена деградація SoH + просадка напруги під навантаженням    |
| `encoder_drift`      | Випадковий шум в енкодерах → одометрія "пливе"                    |
| `brake_stuck`        | Праве колесо частково заклинене (30% потужності)                  |

За замовчуванням:
* AMR-02 має постійну несправність `bearing_right` (знос підшипника);
* AMR-03 має постійну деградацію батареї (`fault_battery_fade`, SoH 83%).

Несправності можна вмикати/вимикати з UI (RobotDetail → "Інжекція
несправностей"), якщо бекенд із MQTT запущено.

## MQTT-топіки

```
factory/line_1/{amr_01|amr_02|amr_03}/telemetry/battery
                                     /telemetry/motors
                                     /telemetry/position
                                     /telemetry/imu
                                     /telemetry/environment
                                     /telemetry/mission
                                     /telemetry/faults
factory/line_1/{amr_xx}/commands          ← inbound: stop/resume/inject_fault
```
